Функционал:

# 1. Система профилей и онбординг :

— Регистрация пользователя происходит по `telegram_id` через Telegram.  
— При первом входе пользователь вводит никнейм и выбирает аватар из доступных вариантов.  
— Никнейм валидируется на сервере: не может быть пустым, слишком длинным или содержать недопустимые символы.  
— Данные профиля сохраняются в БД и привязываются к Telegram UserID.  
— Онбординг показывается только один раз, после первой регистрации.  
— После регистрации пользователь проходит стартовый тест из 10 вопросов разной сложности.  
— По результатам стартового теста сервер определяет стартовый ELO и открывает пользователю соответствующий уровень сложности.  
— Если пользователь уже проходил регистрацию и онбординг, при повторном входе он сразу попадает в основной интерфейс.

# 2. Игровая механика:

— Daily Streak (Огоньки): Счетчик последовательных дней активности. Если пользователь пропустил день — стрик сбрасывается.
— Daily Streak обновляется только если у пользователя была валидная активность в течение календарного дня.
— Система жизней: 3 «сердечка» на тренировку. Ошибка = минус жизнь. 0 жизней = провал тренировки. При провале тренировки сервер применяет соответствующие изменения ELO и сохраняет результат в истории матчей.
— Кнопка A/B: Три кнопки — A слева, Референс посередине, B справа. Пользователь слушает референс сколько угодно раз, потом слушает A и B, выбирает что совпадает или отличается по заданию. Оба аудиобуфера загружаются в память заранее и воспроизводятся синхронно в loop. Переключение через Web Audio API gain node — мгновенное, без перезапуска воспроизведения. Позиция воспроизведения общая для обоих треков. Первый запуск только по нажатию кнопки пользователем (ограничение мобильных браузеров).
Система энергии:
— Пользователь играет тренировочные сессии, состоящие из отдельных упражнений.  
— Каждое упражнение проверяется на сервере, без передачи правильного ответа на фронтенд.  
— Если ответ пользователя пришёл менее чем через 2000 мс после старта упражнения, результат не засчитывается.  
— Минимальное время ответа и другие античит-ограничения проверяются на сервере.  
— У пользователя есть система энергии: одно упражнение расходует энергию, а при достижении нуля новые упражнения недоступны.  
— Максимальный запас энергии, скорость восстановления, стоимость одного упражнения и лимиты для бесплатных пользователей настраиваются через админку.  
— Энергия восстанавливается в реальном времени. Пользователь видит текущий процент восстановления и прогноз до полного восстановления.
— 100% энергии = 20 упражнений (настраивается в админке)
— При 0% энергии пользователь не может проходить упражнения
— Восстановление: от 0 до 100% за 8 часов непрерывно (то есть за 1 час восстанавливается ~12.5% = ~2.5 упражнения). Восстановление идёт в реальном времени, пользователь видит сколько энергии уже накопилось
— Премиум пользователи: безлимитная энергия (настраивается в админке)
Система премиума:
— В таблице users предусмотреть поле is_premium (true/false) и поле premium_until (дата окончания)
— Множитель начисления ELO для премиум пользователей настраивается в админке (например x1.5 или x2)
— Все параметры баланса (лимит энергии, скорость восстановления, множитель ELO, лимит упражнений для бесплатных) должны быть вынесены в настройки и меняться через админку без изменения кода
— Подозрительная активность, слишком частые попытки и аномально быстрые ответы должны помечаться в БД.

# 3. Режимы тренировок

1) Режим «Турбо»: Рандомный микс из всех типов упражнений. Повышенное начисление ELO.  
2) EQ Sniper: Окно спектра 20Hz–20kHz. Логика: клик по области. Подсветка «зоны попадания» (радиус зоны попадания сужается с ростом уровня)
3) Dynamics Detective: Распознавание компрессии (Attack/Release/Threshold).
4) Stereo Scout: Определение положения источника (L/R) или ширины стереобазы.
5) Gain Watcher: Угадывание изменения громкости (например, +/- 1.5 dB).
6) Quality Check: Отличие Lossless от MP3 разного битрейта или поиск клиппинга.


# 4. Система тегов, статусов и профиль пользователя

Страница профиля отображает:
— Аватар и никнейм
— Текущий ELO и ранг
— Daily Streak (количество дней)
— Теги/статусы (несколько одновременно, каждый своим цветом)
— Общая статистика: сыграно сессий, средняя точность по каждому модулю, любимый режим
— История изменения ELO (график)
— Полученные ачивки

Три типа тегов:

1) *Ранговые (автомат, зависят от ELO):*
Новичок / Любитель / Продвинутый / Профи / Эксперт / Легенда — порог ELO для каждого задаётся в админке

2) *Ачивки (автомат, по триггерам):*
— «Острый слух» — 10 идеальных сессий в EQ Sniper подряд
— «Железные уши» — пройти Quality Check без ошибок 5 раз
— «Стрикер» — стрик 30 дней
— «Скоростной» — завершить Турбо режим менее чем за 60 секунд
— и другие (список расширяемый и редактируемый через админку)

3) *Ручные (назначает администратор через админку):*
— Тестер / Администратор / Модератор / VIP / и любые кастомные

Логика отображения:
— В профиле показываются все теги пользователя одновременно, каждый своим цветом (ранговый — золотой, ачивки — синие, ручные — красные/серые)
— Рядом с ником в чате/рейтинге показывается только самый приоритетный тег (ручной > ранговый > ачивка)

# 5. Адаптивное обучение и Статистика

— Smart Focus: Алгоритм анализирует ошибки. Если у пользователя провал в «Компрессии», приложение чаще подкидывает эти задачи в «Турбо» и помечает модуль как «Рекомендуемый».
— Post-Match Analytics: Экран после тренировки:
Логика отображения данных зависит от выбранного режима:
1) Если режим «Турбо» (Смешанный):
Сводная таблица по категориям: Показывает точность (%) отдельно для каждого типа задач, которые встретились в матче (например: EQ Sniper: 80%, Dynamics: 40%, Stereo: 90%).
2) Если выбран конкретный модуль (например, только EQ Sniper):
Процент точности: Крупный индикатор попаданий.
Динамика сессии: Сравнение результата с твоим средним показателем в этом модуле.


# 6. Лидербород

— В приложении есть отдельный экран рейтинга пользователей.  
— Пользователи отображаются в порядке убывания ELO.  
— Рядом с каждым никнеймом показывается один тег с самым высоким приоритетом.  
— Пользователь видит свою текущую позицию в общем рейтинге.  
— Данные лидерборда обновляются на сервере и берутся из актуального состояния БД.



# 7. Админ-панель

— Веб-интерфейс админа доступен по отдельной защищённой ссылке и требует авторизации.  
— Вход в админку должен быть защищён серверной проверкой, а все действия администратора должны логироваться в БД. Не требует командной строки — всё управление через визуальный интерфейс.
